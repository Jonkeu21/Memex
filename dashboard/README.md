# Memex dashboard

FastAPI backend + React (Material UI v5 with a Minimal-UI-Kit-style theme)
frontend, packaged in a single container. Six pages: queue monitoring,
`_inbox/` triage, taxonomy editor, captures browser, rate-limit telemetry,
and retrieval chat.

The dashboard is the operator's primary workstation surface for the Memex
stack on the Pi. It is single-user, exposed only over Tailscale, and gates
all mutating actions behind a single shared bearer token.

## Running it

### Compose (production)

```sh
# From the repo root
docker compose -f infra/docker-compose.yml up -d dashboard
```

The dashboard publishes on `0.0.0.0:8002`, but the Pi has no public ingress —
only the tailnet reaches it. Visit `http://<pi-tailscale-host>:8002/` from a
machine on the tailnet, paste the bearer token into Settings, and start using
it.

The `MEMEX_DASHBOARD_BEARER_TOKEN` value lives in `infra/.env` (generated by
`scripts/bootstrap.sh` with `openssl rand -hex 32`). The bootstrap summary
prints it once when the stack first comes up.

### Backend dev

```sh
# From dashboard/
pip install -e .[dev]
export MEMEX_DASHBOARD_BEARER_TOKEN=dev-token
export MEMEX_DASHBOARD_VAULT_DIR=/srv/memex/vault
export MEMEX_DASHBOARD_DB_PATH=/srv/memex/data/memex.db
uvicorn backend.app:create_app --factory --host 127.0.0.1 --port 8002 --reload
```

### Frontend dev

```sh
cd frontend/
npm install --legacy-peer-deps
npm run dev      # → http://localhost:5173/, proxies /api/* to :8002
```

Vite's dev server proxies `/api/*`, `/healthz`, and `/readyz` to the
backend on `127.0.0.1:8002`. Run the backend first.

## What each page does

| Page | Purpose | Mutations? |
| --- | --- | --- |
| **Queue** | Live queue rows from `memex.db`. Filter by status, drill into a detail drawer with timing + telemetry, retry / cancel. | retry, cancel (token-gated) |
| **Inbox** | Files sitting in `<vault>/_inbox/`. Each row shows title + suggested taxonomy + reason-for-inbox. Side panel renders the markdown; quick-route buttons populate from live `taxonomy.yml`. | route, trash (token-gated) |
| **Taxonomy** | Edit `<vault>/_meta/taxonomy.yml` as a tree. Add/remove/reorder folders, set per-folder threshold overrides. The worker picks up changes on its next batch tick. | save (token-gated) |
| **Captures** | Browse the vault — sortable, searchable, filter by folder + needs_review. Row click opens a markdown viewer. | none |
| **Rate limit** | Reads `claude_calls`. Half-donut gauge for 24-hour total, per-hour stacked bars by service, recent-calls table. | none |
| **Retrieval chat** | Single-shot chat: ask, see loading skeleton, render answer + source chips + excerpts. Sources are clickable; secondary "Open in Obsidian" uses `obsidian://`. | retrieval (token-gated) |

## Auth model

- **Read-only routes** (`GET /api/v1/queue`, `GET /api/v1/inbox`, ...) are
  open on the tailnet. Tailscale is the trust boundary.
- **Mutating routes** (`POST /api/v1/queue/{id}/retry`, `POST
  /api/v1/inbox/{path}/route`, `PUT /api/v1/taxonomy`, `POST
  /api/v1/retrieval`, ...) require `Authorization: Bearer <token>` where
  `<token>` matches `MEMEX_DASHBOARD_BEARER_TOKEN`.
- Comparison is constant-time via `secrets.compare_digest`.
- The frontend stores the token in `localStorage`; clearing it disables
  every mutating control with a tooltip pointing at the Settings drawer.

There is no login screen, no user model, no role/permission scheme. The
dashboard is single-user.

## Path safety

Every vault path that arrives in a path parameter is run through
`backend.vault.safe_join` or `safe_join_existing`, which:

1. Reject NUL bytes, Windows drive letters, and non-string inputs outright.
2. Strip leading slashes (so `"/etc/passwd"` becomes `"etc/passwd"`).
3. Reject any path part equal to `..`.
4. Resolve via `Path.resolve()` and require the result to be relative to the
   vault root — symlinks are followed, and a symlink that escapes the vault
   is rejected.

`tests/backend/test_vault_safety.py` exercises a battery of malicious paths
(`../`, absolute, NUL, drive letter, Unicode, symlink escape, repeated
slashes). That suite was authored before `vault.py` per the prompt's
ordering.

---

## Trash, don't unlink

The inbox "delete" action moves the file to
`<vault>/_trash/<YYYY-MM>/<filename>`, never `os.unlink()`. The vault is the
operator's irreplaceable knowledge — the trash is a one-way valve, not a
shred bin. The UI tooltip says so.

If two files end up with the same name in the same month, the second one is
disambiguated to `<stem>-2.md`.

## Testing

### Backend

```sh
# From dashboard/
python -m pytest tests/backend/
```

Covers ≥ 85% of `backend/`, **184 tests** at the time of writing
(actual: 88.88%). Highlights:

- **Auth**: every mutating route is tested for 401 (no token), 401 (wrong
  token), and 200 (correct token). A static-source check confirms
  `secrets.compare_digest` is used (timing tests are unreliable in CI).
- **Path safety**: 26-test battery of malicious inputs, including
  symlink-escape (skipped on Windows).
- **Inbox routing**: routing moves the file atomically, clears
  `needs_review`, refuses unknown taxonomy targets, and disambiguates
  collisions.
- **Taxonomy**: parser rejects bad YAML, dotdot paths, threshold inversions,
  excessive depth, and duplicate paths. A failed `PUT` does not corrupt the
  on-disk file.
- **Retrieval**: `claude_runner.invoke` is mocked via
  `app.state.claude_runner_invoke`. Timeout / not-authenticated /
  malformed-JSON / transient failures map to 504 / 503 / 502.
- **Rate-limit**: returns `available: false` when the table is missing,
  populated buckets / breakdowns / 5-minute error rate when present.

## Known limitations

- **No real e2e suite.** The Vitest happy path covers the retrieval flow
  against a mocked API, but a Playwright run against a live backend is
  deferred (see `Progress-Tracker.md` Phase 6). The `retrieval.e2e.spec.ts`
  file is a placeholder.
- **`tests/frontend/` lives outside `frontend/`** to match the prompt's
  expected layout. This requires a `dashboard/node_modules` symlink to
  `dashboard/frontend/node_modules` for vitest's module resolution. The
  symlink is git-ignored; create it after `npm install` with:

  ```sh
  ln -s frontend/node_modules dashboard/node_modules
  ```

- **Dashboard frontend bundle** is ~165 kB gzipped for the main chunk +
  ~158 kB for charts (lazy-loaded) + ~50 kB for markdown (lazy-loaded).
  Well under the < 1 MB target for the initial paint.
- **The `obsidian://` URL scheme** requires Obsidian to be installed on the
  client machine viewing the dashboard. The fallback is the in-app markdown
  viewer; both are rendered unconditionally.

## Operator runbook

**Dashboard won't start (HTTP 500 on /readyz):**
Check `docker compose logs dashboard`. The most likely causes are:
1. `MEMEX_DASHBOARD_BEARER_TOKEN` not set (process exits at startup).
2. Vault bind-mount is missing or read-only (the worker also needs it
   read-write — fix in `docker-compose.yml` or `infra/.env`).

**Retrieval returns "Claude CLI not authenticated":**
The `claude_auth` named volume doesn't have credentials. Run:

```sh
docker compose -f infra/docker-compose.yml run --rm --no-deps \
  --entrypoint /usr/local/bin/claude worker /login
```

That same volume is mounted into worker, telegram_bot, and dashboard, so a
single login covers all three.

**Mutating actions silently disabled in the UI:**
Check the Settings drawer — the bearer token field is empty. Paste the
value of `MEMEX_DASHBOARD_BEARER_TOKEN` from `infra/.env`.

**Dashboard "Recent captures" page is slow on a big vault:**
The endpoint walks the PARA folders + `_inbox/` synchronously. On a vault
with > 10 k files this becomes noticeable. Pagination is server-side
(default 100 per page); add a `q=` filter to narrow down before scrolling.
